# DWH | Discord WebHook
DWH is a powerful but simple tool for sending embeds to Discord webhooks.


## Features
#### Send embeds to Discord webhooks with the following customization options:
- Title
- Message
- Author
- Icon URL
- Color
- Footer
#### Also:
- easy integration into your own scripts
- advanced input and error handling
## Usage/Examples
### Command:
```bash
dwh <arguments> <message>
```
#### Arguments:
- ``-h`` show help message
- ``-U`` <discord webhook url> **[REQUIRED]**
- ``-t`` <title>
- ``-a`` <author>
- ``-i`` <icon url> only with author
- ``-c`` <color HEX> without # / default: red
- ``-f`` <footer text>
### Example 1
```bash
dwh -U https://discord.com/api/webhooks/1234567890/ABCDEFGHIJKLMNOPQRSTUVWXYZ This is an example.
```
![Example 1](https://i.imgur.com/O6jTsQY.png)
### Example 2
```bash
dwh -U https://discord.com/api/webhooks/1234567890/ABCDEFGHIJKLMNOPQRSTUVWXYZ -t Title This is an example message with title.
```
![Example 2](https://i.imgur.com/mnafRFe.png)
### Example 3
```bash
dwh -U https://discord.com/api/webhooks/1234567890/ABCDEFGHIJKLMNOPQRSTUVWXYZ -t ":x: Server crashed" -a "Server monitor" -i https://i.imgur.com/YqBOej0.png -c ff0000 -f "Server restart has been initiated." Server XY just crashed.
```
![Example 3](https://i.imgur.com/BKG1BPM.png)

## Installation
```bash
sudo wget https://github.com/retpircs/dwh/master/dwh -O /bin/dwh && chmod +x /bin/dwh
```
If the path for the system commands is not ``/bin``, change it.

## Integration
Once **dwh** is properly installed, you can call the ``dwh`` command in your scripts.
To avoid having to specify all the arguments each time, you can add this configuration to the top of your code:
```bash
url=""
title=""
author=""
icon=""
color=""
```
The config values are filled in automatically when the command ``dwh`` is called. If the default values need to be changed occasionally, you can overwrite them with the appropriate arguments.
It is **not necessary** to insert the variables in your script. However, if you have a lot of embeds, it may become clearer.

## FAQ
#### When do I have to use quotation marks?
Quotation marks must be used for all arguments that have more than one string. An exception to this is ``<message>``.
Example:
```bash
dwh -U https://discord.com/api/webhooks/1234567890/ABCDEFGHIJKLMNOPQRSTUVWXYZ -t string -a "more than one string" <message>
```
If you are unsure, put all arguments in quotation marks.
#### I receive warnings, is that bad?
Yes and no. The good thing is: warnings from DWH do not affect whether an embed is sent. However, an argument will be incorrect and accordingly will not appear in the embed.
Check the warning message and then check the corresponding argument for errors.
